不对称数字系统：将Huffman编码速度与算术编码压缩率相结合的熵编码

发表于 2019-09-17 阅读次数： Valine：
本文字数： 2.1k 阅读时长 ≈ 2 分钟
Jarek Duda
普渡大学信息科学中心，W。Lafayette，IN 47907，U.S.A。
电子邮件：dudaj@purdue.edu

¶摘要

现代数据压缩主要基于两种熵编码方法：霍夫曼（HC）和算术/范围编码（AC）。前者的速度要快得多，但用2的幂来近似概率，通常导致压缩率相对较低。后者使用几乎精确的概率-很容易接近理论压缩率限制（香农熵），但代价要大得多。
非对称数字系统（ANS）是一种精确熵编码的新方法，它允许在速度和速率之间进行权衡：最近的实现[1]提供比256大小字母表的HC快50％的解码速度，压缩率类似于 由AC提供。 这个优点是由于比AC简单：使用单个自然数作为状态，而不是两个来表示范围。 除了简化重整化之外，它还允许将给定概率分布的整个行为放入相对较小的表中：定义熵编码自动机。 256个字母表的这种表的内存成本是几千字节。 选择特定表时有很大的自由度 - 使用用加密密钥初始化的伪随机数生成器可以同时加密数据。
本文还介绍并讨论了这种新的熵编码方法的许多其他变体，它可以为标准AC，大字母范围编码或近似准算术编码提供直接替代方案。

¶介绍

在标准数字系统中，不同的数字被视为包含相同数量的信息：二进制系统中的1比特，或者通常是基数b数字系统中的lg(b)\lg (b)lg(b)比特（lg≡log2\lg \equiv \log _{2}lg≡log​2​​）。 但是，概率p的事件包含lg(1/p)\lg (1 / p)lg(1/p)位信息。 因此，虽然标准数字系统对于统一数字概率分布是最佳的，但为了对一般分布进行最佳编码，数据压缩的核心是什么，我们应该尝试以某种方式使这一概念不对称。 我们可以通过这种方式获得算术/距离编码（AC）（[10]，[9]）或最近的非对称数字系统（ANS）（[2]，[3]），这取决于我们添加后续数字的位置。
具体地，在标准二进制数字系统中，具有以自然数x（具有m个数字）存储的信息，我们可以基本上以两种不同的方式从数字s∈{0,1}添加信息：在最重要的位置（x→ x + 2 ms）或最不重要
位置（x→2x + s）。 前者意味着新数字在大范围之间选择 - 我们可以通过改变这些范围的比例来对其进行非对称化，从而获得算术/范围编码。 然而，在标准数字系统中，我们需要记住这个最高有效位（m）的位置，AC需要在给定时刻指定范围 - 其当前状态是表示范围的两个数字。
相反，在最低有效位置添加信息时，当前状态只是一个自然数。 这种优势仍然存在于非对称化：ANS。 在标准二进制系统x中，x成为相应的偶数（s = 0）或奇数（s = 1）的第x个外观，这次我们想重新定义N到偶数和奇数的分裂。 这样它们仍然是“均匀分布”的，但具有不同的密度 - 对应于我们想要编码的符号概率分布，如图1所示。
让我们从信息论的角度来看两种方法。 对于AC，知道我们当前处于给定范围的一半是值1位信息，因此当前内容是lg（子范围的大小/大小）信息位。 在编码概率p的符号时，子范围的大小乘以p，如预期的那样将信息内容增加lg（1 / p）比特。 从ANS的观点来看，将x视为包含lg（x）位信息，信息内容应增加到lg（x）+ lg（1 / p）= lg（x / p）位，同时添加概率p的符号。 所以我们需要确保x→≈x/ p过渡关系。 直观地，规则：x成为第s个子集的第x个外观，如果该子集以Pr（s）密度均匀分布，则满足该关系。

图1：两种不对称二进制数字系统的方法。 将一些信息存储在自然数x中，以附加来自0/1符号s的信息，我们可以将其添加到最重要的位置（x 0 = x + s2 m），其中s在范围之间选择，或者在最不重要的位置选择 （x 0 = 2x + s）位置，其中s在偶数和奇数之间选择。 前者通过改变范围比例而不对称于AC。 后者通过重新定义偶数/奇数来不对称于ABS，使得它们仍然均匀分布，但具有不同的密度。 现在x 0是第s个子集的第x个元素。

图2：左：霍夫曼编码的构造，同时将（1,1,1）/ 3概率分布中的两个符号分组：我们有9个等概率可能性。 该分组减少了与香农熵的距离：ΔH。 标准基础3数字系统在这里给出ΔH= 0。 右：ΔH用于分组m个符号。 例如，在3 10 = 59049的情况下操作可以获得损失ΔH≈0.003位/符号。 相比之下，对于最后四种情况仅使用16或17个状态的ANS的ΔH相应地：≈0.00065,0.00122,0.00147,0.00121位/符号。
